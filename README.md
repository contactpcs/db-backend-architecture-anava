# Anava Clinic — Backend v2

Neurological care platform backend for Anava Clinic (Mana Health Sciences Group). Rewrite of the NeuroWellness FastAPI backend onto an AWS-native stack (RDS, Cognito, S3) with new Store/Payments modules.

## Current status

Stages 0-12 of the 22-stage development plan are built and verified against a real local Postgres database (Docker). Working end-to-end: patient registration → consent → anamnesis → PRS scoring → doctor auto-allocation → clinical sessions (S1-S4) → treatment plans → extended-session payment gating → appointment scheduling with double-booking prevention → device/accessory store orders → inventory dispatch → live SSE notifications → follow-up cycles → clinic transfers → patient exit.

14 of 16 modules are implemented: `auth`, `admin`, `staff`, `patients`, `clinical`, `scheduling`, `prs`, `anamnesis`, `files`, `consent`, `store`, `inventory`, `payments`, `notifications`. Not yet built: `audit` (read-side query API), `reports`.

Everything runs locally against Docker services (Postgres/Redis/ElasticMQ) — no AWS account or Razorpay keys required yet. Real AWS/Cognito/S3 and live Razorpay wiring is Stage 13, not started.

## Quickstart

See `backend/README.md` for full setup. Short version:

```bash
cd backend
python -m venv .venv
.venv/Scripts/pip install -e ".[dev]"     # Windows; use .venv/bin/pip on Linux/Mac
cp .env.example .env
docker compose up -d
alembic upgrade head
python -m scripts.seed_dev_profile
uvicorn app.main:app --reload
```

Then `POST /api/v1/auth/local-login {"cognito_sub": "dev-super-admin"}` to get a token — no Cognito account needed for local dev.

## What's gitignored and why

- `backend/.venv/`, `__pycache__/`, build artifacts — machine-specific, regenerated by `pip install`.
- `backend/.env` — local secrets (dev mode uses a placeholder JWT secret only, never real credentials; `.env.example` is the template that IS committed).
- `backend/.local_storage/` — local filesystem stand-in for S3, dev-only file uploads.
- Standard OS/IDE noise (`.DS_Store`, `Thumbs.db`, `.vscode/`, `.idea/`).

No real credentials, API keys, or PHI-shaped data belong in this repo at any point — local dev uses synthetic seed data only (`scripts/seed_dev_profile.py`, `scripts/seed_test_prs_content.py`).
